According to my Thinkful mentor Walter G. in a recent article on his slick site thatsoftwaredude.com, “Whether you are just getting started with a programming language and a tech stack, or if you have 10 years of experience already and building complex scalable apps, you need to have a blog.” After reading his article, I agreed, and began work on this blog.

I chose Jekyll as my blogging platform for the following reasons:

  • Coding a blog from scratch would be too hard
  • Using a WYSIWYG editor like Blogger would be too easy
  • Jekyll is just right
    • I can write posts in Markdown on any text editor
    • I can edit HTML & CSS source code, with default themes serving as scaffolding for me to improve my skills
    • All files exist in a local repo, and Jekyll can serve my site locally with live reload, so I can see how changes to source code affect my site in real-time
    • Free, integrated web hosting on GitHub Pages, with custom domain name

That said, the process of installing prerequisite programs (Ruby, RubyGems, & Bundler), installing Jekyll, creating a default site, & pushing my local repo to GitHub was waaaaaay harder than I thought it would be and worthy of its own post.
